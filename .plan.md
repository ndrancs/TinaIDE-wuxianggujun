# Linux 发行版下载：远程清单 + 镜像回落 实现计划

## 目标（顺着你的思路：数据挂到 registry，更新数据≠发版）

把"有哪些发行版、从哪下、有哪些镜像"这份数据，从**写死在 App 资源里**，改成**托管在 `TinaIDE-Registry` 仓库**（和 `plugins/index.v2.json`、`packages/index.v2.json` 同一套机制）。两个收益一次拿到：

- **P0 远程清单**：上游再改路径（今天的 404）、要加发行版、改版本 → 改远端 JSON 即可，用户无需更新 App。
- **P1 镜像回落**：每个下载源带镜像规则；下载失败自动切清华/中科大/阿里云等国内镜像。

安全底线：**远程拉取永远是"增强"，任何失败都回落到内置 asset**（今天已修好的 `v3.23` 清单），所以即使 registry 挂了、网络断了，安装仍能走内置数据完成。

## 已确认的关键事实（逐文件核对）

- `:core:linux-distro` 是**底层模块**，只依赖裸 okhttp，**不依赖 `:core:network`** → 保持网络无关、可测。
- `:core:proot` 同时依赖 `linux-distro` + `network` → **远程拉取逻辑落在这一层**（装配层）。
- 清单加载只有两处构造 `AndroidAssetLinuxDistroManifestSource`：
  - `SelfHostedLinuxDistroRuntime.createFromAssets()`（proot，行 245）
  - `RootfsDistroRuntime.listDistros()` / `installDistro()`（proot）
- 下载入口：`LinuxDistroInstaller.ensureArchive()`（行 162-185）当前用单个 `resolved.artifact.url` 下载一次。
- `DistroArtifact` 只有单 `url` 字段，无镜像。
- registry 已有多端点回落：GitHub Raw + jsDelivr + 4 个国内代理前缀（`GitHubRegistryConfig`）。
- 镜像路径已联网验证可达（200）：tuna / ustc / aliyun / nju 的 `alpine/v3.23/...`。

## 设计要点（KISS / DRY / 分层）

### 1. 镜像用"前缀替换规则"，不逐条列全 URL（DRY）

逐 artifact 列 4 个镜像 × 4 架构 × 2 发行版 = 几十行重复。改用**清单级前缀替换规则**：

```json
"mirrors": [
  { "matchPrefix": "https://dl-cdn.alpinelinux.org/", "replaceWith": "https://mirrors.tuna.tsinghua.edu.cn/" },
  { "matchPrefix": "https://dl-cdn.alpinelinux.org/", "replaceWith": "https://mirrors.ustc.edu.cn/" }
]
```

一条规则覆盖某发行版全部架构。artifact 的 `url` 保持官方规范地址，installer 按规则派生候选。维护者改远端 JSON 加一行即可加镜像。

### 2. 真正的层次归属

- **`:core:linux-distro`（纯数据/逻辑，可单测）**：加 `mirrors` 模型 + installer 多候选下载。不碰网络。
- **`:core:proot`（装配层，有 network）**：新增"远程清单源"——远程拉取 → 磁盘缓存(带 TTL) → 内置 asset 回落。

### 3. 缓存策略（避免每次开设置页都打网络）

清单源构造频繁（settings 列表会调 `listDistros`）。规则：
- 缓存新鲜（< TTL，默认 6h）→ 直接用缓存。
- 缓存过期 → 尝试远程（短超时）→ 成功则写缓存并用；失败 → 用过期缓存；再无 → 用内置 asset。

## 改动清单

### A. 镜像数据模型 + 多候选下载（`:core:linux-distro`，纯逻辑）

**A1. 模型** — `LinuxDistroModels.kt` / `LinuxDistroManifest.kt`
- 新增 `@Serializable data class DistroMirrorRule(val matchPrefix: String, val replaceWith: String)`。
- `LinuxDistroManifest` 加 `val mirrors: List<DistroMirrorRule> = emptyList()`（清单级，默认空 → 向后兼容旧 asset）。

**A2. catalog 暴露镜像规则** — `LinuxDistroCatalog.kt`
- 接口加 `fun mirrorRules(): List<DistroMirrorRule> = emptyList()`。
- `ManifestLinuxDistroCatalog` 实现返回 `manifest.mirrors`。

**A3. installer 多候选回落** — `LinuxDistroInstaller.kt`（`ensureArchive`）
- 用 `resolved.artifact.url` + `catalog.mirrorRules()` 派生候选列表：
  规范 url 先试，再按规则 `replaceWith + url.removePrefix(matchPrefix)` 追加镜像候选（去重）。
- 循环候选 `download()`：成功即停；非取消异常 → 记下并试下一个；`CancellationException` → 立即 rethrow（不试镜像）。全部失败 → 抛最后一个异常。
- 校验和（行 60）保持在循环外执行一次（镜像同内容；极少数损坏直接判失败，YAGNI 不做逐镜像校验）。

### B. 远程清单源（`:core:proot`，装配层）

**B1. 新文件** `core/proot/.../RemoteLinuxDistroManifestSource.kt`
- 实现 `LinuxDistroManifestSource`（接口在 linux-distro，已 public）。
- 组合逻辑：缓存(TTL) → 远程（走 `GitHubRegistryConfig` 多端点候选 + `OkHttpClientProvider.probe`）→ asset 回落。
- 远程解析用现成的 `LinuxDistroManifestParser.decode`，**整段 `runCatching`**：远程 schema 比 App 新、JSON 坏、网络错 → 全部静默回落到 asset（不崩、不打断安装）。
- 缓存文件放 `SelfHostedLinuxDistroRuntime.defaultRuntimeDir(context)` 下，如 `linux-distro/manifest.cache.json` + 时间戳。

**B2. registry 路径常量** — `GitHubRegistryConfig.kt`
- 加 `const val LINUX_DISTRO_MANIFEST_PATH = "linux-distro/manifest.v1.json"`，复用现有 `registryEndpoints()` / `resolveRawUrl()` 生成候选 URL（自动带 jsDelivr + 国内代理回落）。

**B3. 接线** — proot 两处构造点改用新源
- `SelfHostedLinuxDistroRuntime.createFromAssets()`（行 245）：`AndroidAssetLinuxDistroManifestSource(...)` → `RemoteLinuxDistroManifestSource(context, fallback = AndroidAsset...)`。
- `RootfsDistroRuntime.listDistros()` / `installDistro()`：同样替换。
- `AndroidAssetLinuxDistroManifestSource` 保留，作为回落委托，不删。

### C. 发布远程清单到 registry（数据，不在本仓库）

- 提供一份 `linux-distro/manifest.v1.json`：内容 = 当前已修好的 asset 清单（`v3.23` 路径）+ 新增 `mirrors` 规则（alpine/ubuntu 各配 tuna/ustc/aliyun）。
- 我生成文件内容；**你 push 到 `TinaIDE-Registry` 仓库 `main` 分支**（跨仓库，本任务不直接推）。
- push 后即时生效，老版本 App 也能拉到（只要 App 已含 B 的逻辑）。

## 不做什么（YAGNI 边界）

- 不把 `:core:linux-distro` 升级为依赖 `:core:network`（保持底层纯净）。
- 不做"镜像优先/官方优先"开关（默认官方先、镜像后回落；够用）。
- 不做自定义 rootfs URL 导入（高级特性，后续单独迭代）。
- 不改首次引导流程的"写死 Alpine"（那是发行版**选择 UI**，属另一条 P2，本次只解决"下得到/下得快/可远程更新"）。
- 不做手动"刷新清单"按钮（TTL 自动管理；如需再加）。
