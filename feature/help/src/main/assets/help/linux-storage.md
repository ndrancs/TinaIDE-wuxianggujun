# Linux 系统与存储

本文基于当前“存储”设置页源码整理，重点说明 rootfs 导入、配置档案、MT 访问和存储清理这些与开发环境最相关的入口。

## 存储页现在管什么

当前存储页主要分成三块：

- **Linux 系统**
- **高级**
- **存储清理**

其中“Linux 系统”部分只有在当前启用了 Linux 开发环境后才会显示。

## Linux 系统区里有什么

当前页面会显示这些入口：

- 从预设导入 rootfs
- 从本地导入 rootfs
- 从 URL 导入 rootfs
- 当前 Linux 系统
- Linux 发行版安装入口
- Linux 配置档案列表
- 导入状态

如果你要排查“环境装在哪、当前到底在用哪个系统、导入为什么失败”，这里是第一入口。

## rootfs 预设导入

如果当前设备上有可用预设目录，页面会显示：

- **导入预设**

这类预设本质上是应用内提供的 rootfs 描述集合，适合：

- 想少填参数
- 想按推荐方式快速导入
- 不想自己找下载链接

如果你没有看到这个入口，通常说明当前没有可用预设列表，不一定是异常。

## 从本地导入

当前页面支持通过系统文档选择器导入本地文件。

适合场景：

- 你已经有本地 rootfs 包
- 从电脑拷贝到了设备上
- 想离线导入

这条路径最大的价值是可控，不依赖网络。

## 从 URL 导入

当前页面也支持直接输入下载地址导入。

### 当前校验规则

源码里要求 URL 至少满足：

- 以 `http://` 或 `https://` 开头

也就是说，如果你填的是：

- 裸域名
- 不带协议的地址
- 其他奇怪前缀

页面会直接判成无效输入。

## 当前 Linux 系统

页面会显示：

- **当前 Linux 系统**

它展示的是当前激活的 rootfs 配置档案名称。如果没有可用系统，则会显示未安装状态。

这是判断“当前终端到底进的是哪套系统”的最快方式之一。

## Linux 发行版安装入口

存储页里有 **安装 Linux 发行版**、当前系统和运行状态相关入口。

它会显示：

- 当前 rootfs 的路径
- 当前安装状态

点击后会进入开发环境安装/部署相关流程。

如果你怀疑当前 Linux 发行版没装好，不需要先去终端瞎试，直接先看这里。

## Linux 配置档案是什么

当前实现支持多个 rootfs 配置档案，每个配置档案会显示：

- 显示名称
- rootfs 路径
- 当前状态或包管理器类型

你可以把它理解成：

- 一套 Linux 系统
- 一条 rootfs 目录
- 对应的包管理器识别结果

## 配置档案能做哪些操作

点击某个配置档案后，当前实现可能提供：

- **切换为当前**
- **重命名**
- **删除**

但不是所有配置档案都能做全部操作。

### 受保护档案的限制

源码里明确限制了：

- **受保护 profile 不能重命名**
- **受保护 profile 不能删除**

所以如果你点进去没看到“删除”或“重命名”，这通常是设计使然，不是 UI 漏了。

### 当前正在使用的档案

如果该 profile 已经是当前激活项，就不会再显示“切换”动作。

## 导入状态怎么看

导入过程中或导入完成后，页面会显示：

- 当前导入消息
- 导入进度

如果导入正在进行中，还会出现加载对话框。

因此“点了导入没反应”的时候，先看导入状态区域，而不是立刻重复操作。

## Linux rootfs 路径

高级区域里会显示：

- **Linux rootfs 路径**

如果当前没有自定义路径，会显示默认值；否则会显示当前实际 rootfs 路径。

这个值适合用来确认：

- 环境到底装到了哪里
- 当前是不是还在用默认位置

## MT 访问是什么

高级区域里还有一个开关：

- **MT 访问**

它默认关闭。开启后，仅允许 MT 管理器访问 TinaIDE 自己的四类目录：

- `data`
- `Android/data`
- `Android/obb`
- `user_de_data`

它适合在需要用 MT 管理器处理 TinaIDE 应用目录时临时开启。不要把它理解成开放整机存储或其他应用目录。

如果你遇到：

- MT 管理器看不到 TinaIDE 应用目录
- 需要临时查看或整理 TinaIDE 自己的应用数据

可以检查这里是不是符合你的预期。

## 存储清理页能做什么

存储页底部有一个独立入口：

- **存储清理**

当前清理页支持：

- 扫描总占用
- 重新扫描
- 一键清理全部
- 按分类进入详情查看和清理

### 总览页的真实行为

当前总览页会先扫描所有分类，再显示：

- 各分类占用
- 总占用
- 一键清理全部

如果你正在重新扫描或正在执行一键清理，当前页面也会限制某些操作，避免重复触发。

## 当前清理分类

源码里当前已接入这些分类：

- 构建中间产物
- PRoot 缓存
- 下载缓存
- 导出缓存
- 应用日志
- 安装日志

其中两类日志的实际含义，当前字符串里写得比较清楚：

- **应用日志**
  - 构建日志与 PRoot 会话日志
- **安装日志**
  - PRoot 环境与依赖安装过程日志

如果你只是觉得“应用越来越占空间”，先看这个清理页，比手动猜目录更靠谱。

## 什么时候该清理什么

可以这样理解：

- **构建中间产物**
  - 适合构建很多次后回收空间
- **PRoot 缓存**
  - 适合环境相关缓存积累太多时处理
- **下载缓存**
  - 适合重复下载过很多资源时清理
- **导出缓存**
  - 适合导出过 APK、日志或其他产物后清理
- **应用日志 / 安装日志**
  - 适合排障完成后回收空间

## 详情页现在能做什么

点进某个分类后，当前详情页不是只给一个“全部删除”按钮，而是支持更细的查看和选择：

- 顶层列表查看
- 目录展开 / 收起
- 按需加载子项
- 顶层全选 / 清除选择
- 清理选中项

### 选择逻辑怎么理解

当前实现里：

- 顶层目录使用**三态选择**
- 子项可以单独勾选
- 如果父级整体选中了，就不会再重复按子项累计

也就是说，详情页不是“全有或全无”的粗暴清理，而是可以按目录和文件粒度裁剪。

## 一键清理全部要注意什么

当前清理页支持：

- **清理全部**

这很适合快速回收空间，但不适合你还在排障时立刻点，因为：

- 应用日志
- 安装日志
- 某些缓存线索

可能正是你稍后还要看的东西。

如果你正在查问题，建议先导出日志，再做清理。

### 当前不会清掉什么

从确认文案来看，当前“一键清理全部”不会影响：

- 用户源码
- 工程配置
- 字体

所以它主要针对缓存、中间产物和日志，而不是动你的项目正文。

### 为什么清完后还可能提示失败

当前实现允许“部分文件删除失败”。

所以你可能会看到两种结果：

- 全部成功，直接显示释放空间
- 部分失败，但仍然显示已经释放了多少空间

这不一定说明清理完全没生效，只是有一部分路径没删掉。

## 常见问题

### 为什么存储页没有 Linux 系统区

先检查：

1. 当前是否启用了 Linux 开发环境
2. 当前会话状态是否已刷新

源码里这块内容本来就是按 `linuxEnvironmentEnabled` 条件显示的。

### 导入 URL 为什么一直报无效

先看地址是不是：

- 以 `http://` 开头
- 或以 `https://` 开头

不满足这个条件，页面会直接拦住。

### 为什么某个 profile 不能删除

通常是因为它是：

- 当前正在使用的 profile
- 或受保护 profile

其中受保护 profile 在当前实现里本来就不允许删除。

### 存储空间不够了先去哪里

优先顺序建议：

1. 先进 **存储清理**
2. 看哪些分类占用最大
3. 清理日志、导出缓存、构建中间产物
4. 仍不够时，再考虑重新整理 rootfs 与环境文件
