# 终端与开发环境排障

本文基于当前终端设置页源码整理，重点覆盖后端模式、Shell、开发包、Zsh、Locale 这些最常出问题的点。

## 先理解当前终端设置页

现在的终端设置主要分成这几块：

- **终端主题**
- **字体**
- **Shell**
- **Linux 工具**
- **交互**

如果终端表现异常，通常不要只看“终端打不开”这一层，而是先判断问题属于哪一块。

## 后端模式怎么选

当前实现支持：

- **AUTO**
- **PROOT**
- **HOST**

### 一般建议

- 不确定时：先用 **AUTO**
- 需要完整 Linux 工具链、包管理、调试链路：优先 **PROOT**
- 只想靠近宿主环境、做轻量命令：可以用 **HOST**

### 什么时候会切不过去

如果你切到 **PROOT** 时失败，最常见原因就是：

- PRoot 环境尚未安装
- 开发环境部署不完整

这时候不要一直切模式，先去重新部署开发环境。

## Shell 类型怎么理解

当前可选：

- **AUTO**
- **sh**
- **bash**
- **zsh**

### 建议

- 想要最稳妥：先用 **bash**
- 不确定当前环境状态：先回到 **AUTO** 或 **bash**
- 只有在 Zsh 已安装完成后，再切到 **zsh**

如果你切到 Zsh 后终端打不开，先切回 Bash 验证基础环境。

## Linux 工具区域看什么

这一块是当前终端排障最关键的区域，会显示：

- Linux 工具健康状态
- 当前系统名称
- 当前包管理器
- Guest Dev Packages 状态
- Zsh 状态
- Locale 支持状态

也就是说，这里不仅是“看信息”，还是判断开发环境是否完整的第一现场。

## Guest Dev Packages 是什么

当前实现里，这一项代表一组常用开发命令是否可用，例如：

- 编译器相关命令
- `make`
- `git`
- `curl`
- `cmake`
- `pkg-config` / `pkgconf`

### 什么时候该装它

出现这些情况时，优先检查它：

- 终端里 `cmake` / `make` / `git` 提示找不到
- 工程能打开，但命令行构建完全跑不起来
- 你想在终端里做完整构建流程

### 当前动作

根据状态不同，页面可能提供：

- 安装
- 重新安装
- 刷新状态

如果命令缺失，但你不确定缺哪一项，先走“刷新状态”，再看缺失组件提示。

## Zsh 相关

当前终端设置已经把 Zsh 独立成一个可检查、可安装的能力。

### 常见情况

- **未安装**：先安装，再切换 Shell
- **已安装但不可用**：先刷新 Linux 工具状态，再尝试重新安装
- **HOST 后端下想用 Zsh**：当前实现可能并不总能满足，先回到 PRoot 或 Bash 验证

如果你只是想稳定工作，不要把“换成 Zsh”当成第一优先级。

## Locale 和中文乱码

当前页面同时提供：

- 终端交互语言设置
- Linux Locale 支持安装/重建

### 典型症状

- 中文乱码
- 非 UTF-8 输出异常
- 某些命令显示字符不正常

### 建议顺序

1. 先检查当前 `Locale`
2. 再检查 `Locale 支持` 是否已安装
3. 如果装过但仍有问题，尝试重建 Locale 支持

仅仅改语言值，不一定等于 Linux 环境里真的已经有对应 Locale 数据。

## 字体和显示问题

当前终端支持：

- 主题切换
- 字体大小
- 内置字体
- 系统等宽字体
- 自定义字体
- 光标闪烁与闪烁速率

### 常见排查

- 字符看起来拥挤：调字体大小
- 某些符号显示不好：切换到系统等宽字体或更换自定义字体
- 光标体验不舒服：关掉闪烁或降低频率

如果你导入自定义字体失败，先确认文件本身确实是有效字体文件。

## 终端打不开时先怎么查

建议按这个顺序：

1. 看后端模式是不是切到了不合适的值
2. 检查 PRoot 环境是否已经就绪
3. 进入 Linux 工具区域看健康状态
4. 必要时重新部署开发环境

如果你一上来只盯着终端页面本身，往往会漏掉根因其实在环境没装好。

## 命令找不到时怎么查

例如遇到：

- `cmake: command not found`
- `make: command not found`
- `git: command not found`

优先检查：

1. 当前是不是在 **PROOT** 后端
2. Guest Dev Packages 是否已安装
3. Linux 工具健康状态是否提示缺失

## Zsh 不能用时怎么查

先按这个顺序：

1. 检查当前 Shell 是否真切到了 `zsh`
2. 检查 Zsh 状态是否为已安装
3. 尝试刷新 Linux 工具状态
4. 不行就重新安装 Zsh
5. 仍异常时先回 Bash，验证基础环境

## 中文乱码怎么查

优先检查：

1. 当前 Locale 值
2. Locale 支持是否已安装
3. 是否需要重建 Locale 支持
4. 是否是字体本身不适合显示对应字符

## 什么时候该重新部署开发环境

出现这些情况时，可以直接考虑重新部署，而不是继续微调：

- PRoot 迟迟不可用
- Linux 工具状态长期异常
- Guest Dev Packages 反复安装失败
- LLDB / 终端 / 构建链路一起异常

重新部署入口通常在编译器相关设置里，而不是终端页本身。

## 最后一个实用建议

排障时尽量一次只改一个变量：

- 先改后端模式
- 再改 Shell
- 再装开发包
- 最后再动 Zsh / Locale / 字体

这样最容易看出到底是哪一步解决了问题。
