# 键盘快捷键

本文基于当前源码整理。现在的快捷键系统主要面向 **硬件键盘**，并支持在设置中自定义，而不是桌面 IDE 那种完整的 JSON 键位系统。

## 当前默认快捷键

### 文件操作

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + S` | 保存当前文件 |
| `Ctrl + Shift + S` | 保存全部文件 |

### 标签页

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + W` | 关闭当前标签 |
| `Ctrl + Shift + W` | 关闭全部标签 |
| `Ctrl + Tab` | 切换到下一个标签 |
| `Ctrl + Shift + Tab` | 切换到上一个标签 |

### 编辑

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + Z` | 撤销 |
| `Ctrl + Shift + Z` | 重做 |

### 书签

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + F2` | 切换当前书签 |
| `F2` | 跳转到下一个书签 |
| `Shift + F2` | 跳转到上一个书签 |

## 如何自定义

进入：

**设置 → 快捷键**

当前页面支持这些能力：

- 查看每个动作的当前快捷键
- 点击某个动作开始重新录入
- 检测与现有快捷键冲突
- 单独恢复默认值
- 一次性恢复全部默认快捷键

## 录入新快捷键时会发生什么

设置页会进入“捕获按键”状态。你只需要：

1. 点开要修改的动作
2. 按下新的组合键
3. 确认保存

如果这个组合键已被别的动作使用，应用会提示冲突。

## 使用前请注意

### 需要硬件键盘

当前快捷键系统依赖真实按键事件，因此：

- 蓝牙键盘
- 平板键盘壳
- USB 键盘

会比软键盘更稳定。

### 软键盘不一定可靠

很多软键盘不会完整发出 `Ctrl`、`Shift`、`F2` 这类组合事件，所以不要把它当成主要使用方式。

### 终端和编辑器是两套交互

编辑器快捷键与终端里的扩展按键栏不是一回事：

- 编辑器快捷键：用于保存、标签切换、书签等
- 终端扩展按键：用于补齐 `Ctrl`、`Alt`、`Esc` 等命令行控制键

## 推荐先记住这几个

如果你刚开始用硬件键盘，建议先熟悉：

1. `Ctrl + S`
2. `Ctrl + Z`
3. `Ctrl + Tab`
4. `Ctrl + W`
5. `Ctrl + F2`

这几组最能立刻提升编辑效率。

## 常见问题

### 我按了组合键但没反应

优先检查：

- 是否连接的是硬件键盘
- 当前焦点是否在编辑器中
- 这个组合键是否已被你改过

### 为什么没有桌面 IDE 那种几百个默认快捷键

当前实现是 **聚焦高频动作** 的简化方案，先覆盖保存、标签、撤销重做和书签等常用操作。

### 可以恢复默认吗

可以。设置页支持：

- 恢复单个动作默认值
- 恢复全部默认快捷键
