## 🔴 严重问题（需要立即修复）

### 1. Project 定义重复冲突

**位置**:
- `app/src/main/java/com/wuxianggujun/tinaide/model/DataModels.kt` (完整定义)
- `app/src/main/java/com/wuxianggujun/tinaide/file/IFileManager.kt` (简化定义)

**问题描述**:
两个文件中定义了不同的 `Project` 类，导致类型冲突和编译错误。

```kotlin
// DataModels.kt - 完整的领域模型
data class Project(
    val id: String,
    val name: String,
    val rootPath: String,
    val type: ProjectType,
    val config: ProjectConfig,
    val createdAt: Long,
    val lastModified: Long
)

// IFileManager.kt - 简化的数据传输对象（冲突！）
data class Project(
    val name: String,
    val rootPath: String,
    val files: List<File>
)
```

**影响**:
- FileManager 返回的 Project 类型与其他模块不一致
- 无法在不同模块间传递 Project 对象
- 序列化/反序列化会出错

**修复方案**:

1. **删除 IFileManager.kt 中的 Project 定义**
2. **统一使用 DataModels.kt 中的定义**
3. **修改 FileManager.kt 的实现**

```kotlin
// IFileManager.kt - 修改后
package com.wuxianggujun.tinaide.file

import com.wuxianggujun.tinaide.model.Project  // 导入统一的 Project
import java.io.File

    fun openProject(path: String): Project  // 使用统一的 Project
    fun getCurrentProject(): Project?
    // ... 其他方法

```

```kotlin
// FileManager.kt - 修改后
override fun openProject(path: String): Project {
    val projectDir = File(path)
    require(projectDir.exists() && projectDir.isDirectory)

    closeProject()

    // 尝试从 project.json 加载
    val project = Project.fromDirectory(projectDir).getOrElse {
        // 创建新项目
        Project(
            id = java.util.UUID.randomUUID().toString(),
            name = projectDir.name,
            rootPath = projectDir.absolutePath,
            type = detectProjectType(projectDir),
            config = ProjectConfig(language = "cpp"),
            createdAt = System.currentTimeMillis(),
            lastModified = System.currentTimeMillis()
        )
    }
    
    currentProject = project
    configManager.set(ConfigKeys.CurrentProject, path)
    
    return project
}

private fun detectProjectType(dir: File): ProjectType {
    val files = dir.walkTopDown().filter { it.isFile }.take(100).toList()
    val hasCpp = files.any { it.extension in listOf("cpp", "cc", "cxx") }
    val hasC = files.any { it.extension == "c" }
    
    return when {
        hasCpp && hasC -> ProjectType.MIXED
        hasCpp -> ProjectType.CPP
        hasC -> ProjectType.C
        else -> ProjectType.UNKNOWN
    }
}
```

### 2. Project 缺少派生属性和行为方法

**问题描述**:
当前的 Project 只是一个数据容器，缺少领域模型应有的行为。

**建议添加**:

```kotlin
// 在 DataModels.kt 中添加扩展属性和方法
val Project.root: File
    get() = File(rootPath)

val Project.buildDir: File
    get() = File(AppDirectories.getBuildDir(name))

val Project.sourceFiles: List<File>
    get() {
        val extensions = when (type) {
            ProjectType.C -> listOf("c")
            ProjectType.CPP -> listOf("cpp", "cc", "cxx")
            ProjectType.MIXED -> listOf("c", "cpp", "cc", "cxx")
            ProjectType.UNKNOWN -> listOf("c", "cpp", "cc", "cxx")
        }
        
        return config.sourceDirectories
            .flatMap { srcDir ->
                File(rootPath, srcDir).walkTopDown()
                    .filter { it.isFile && it.extension in extensions }
                    .toList()
            }
            .ifEmpty {
                File(rootPath, "src").walkTopDown()
                    .filter { it.isFile && it.extension in extensions }
                    .toList()
            }
    }

fun Project.clean(): Result<Unit> = runCatching {
    buildDir.deleteRecursively()
    buildDir.mkdirs()
}

fun Project.delete(): Result<Unit> = runCatching {
    if (!root.startsWith(AppDirectories.projectsDir)) {
        throw IllegalStateException("Cannot delete project outside projects directory")
    }
    root.deleteRecursively()
}
```

### 3. FileManager 性能问题

**问题描述**:
`openProject` 方法中使用 `listFiles()?.toList()` 加载所有文件，大型项目会卡顿。

**当前代码**:
```kotlin
val files = projectDir.listFiles()?.toList() ?: emptyList()
```

**建议**:
使用懒加载，只在需要时加载文件列表：

```kotlin
// 不在 openProject 时加载所有文件
override fun openProject(path: String): Project {
    // ... 只加载项目元数据
    val project = Project.fromDirectory(projectDir).getOrThrow()
    currentProject = project
    return project
}

// 文件列表通过扩展属性懒加载
val Project.files: List<File>
    get() = File(rootPath).listFiles()?.toList() ?: emptyList()
```

## 🟢 轻微问题（可选优化）

### 4. 缺少 Prefs 单例门面

**问题描述**:
虽然已有类型安全的 ConfigKey，但每次使用都需要通过 ServiceLocator 获取 ConfigManager。

**当前用法**:
```kotlin
val configManager = ServiceLocator.get<IConfigManager>()
val theme = configManager.get(ConfigKeys.Theme)
```

**建议添加 Prefs 门面**:
```kotlin
object Prefs {
    private val configManager: IConfigManager by lazy {
        ServiceLocator.get<IConfigManager>()
    }
    
    val appTheme: String
        get() = configManager.get(ConfigKeys.Theme)
    
    val currentProject: String
        get() = configManager.get(ConfigKeys.CurrentProject)
    
    fun setTheme(theme: String) {
        configManager.set(ConfigKeys.Theme, theme)
    }
}

// 使用
val theme = Prefs.appTheme  // 更简洁
```

### 5. 缺少 AppDirectories 统一目录管理

**问题描述**:
路径硬编码分散在各处，不便于维护。

**建议**:
创建 `AppDirectories` 对象统一管理所有目录：

```kotlin
object AppDirectories {
    private lateinit var context: Context
    
    fun init(context: Context) {
        this.context = context.applicationContext
        ensureDirectories()
    }
    
    val dataDir: File get() = context.filesDir
    val projectsDir: File get() = File(dataDir, "projects")
    val buildDir: File get() = File(context.cacheDir, "build")
    val sysrootDir: File get() = File(dataDir, "sysroot")
    
    fun getBuildDir(projectName: String): File {
        return File(buildDir, projectName)
    }
    
    private fun ensureDirectories() {
        listOf(projectsDir, buildDir, sysrootDir).forEach {
            it.mkdirs()
        }
    }
}
```
