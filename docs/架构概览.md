## 🏗️ 核心模块

### 1. UI Layer (界面层)

**技术栈**: Jetpack Compose + Material Design 3

**主要组件**:
- `MainActivity.kt` - 主活动
- `ui/screens/` - 各个界面屏幕
- `ui/components/` - 可复用 UI 组件
- `ui/theme/` - 主题和样式

**职责**:
- 用户交互
- 界面渲染
- 状态展示

### 2. Editor Layer (编辑器层)

**技术栈**: Sora Editor

**主要组件**:
- `editor/EditorFragment.kt` - 编辑器容器
- `editor/language/` - 语言支持
- `editor/completion/` - 代码补全

**职责**:
- 代码编辑
- 语法高亮
- 代码补全
- 错误提示

### 3. Project Management (项目管理)

**主要组件**:
- `project/ProjectManager.kt` - 项目管理器
- `project/ProjectType.kt` - 项目类型定义
- `project/template/` - 项目模板

**职责**:
- 项目创建
- 项目打开
- 文件管理
- 项目配置

### 4. Build System (构建系统)

**主要组件**:
- `core/compiler/CompileProjectUseCase.kt` - 编译用例
- `core/compiler/SingleFileCompiler.kt` - 单文件编译器

**职责**:
- 检测项目类型
- 执行编译流程
- 管理构建产物
- 错误处理

### 5. Native Bridge (原生桥接)

**主要组件**:
- `core/nativebridge/NativeLoader.kt` - 库加载器
- `core/nativebridge/SysrootInstaller.kt` - Sysroot 安装器
- `core/nativebridge/NativeCompiler.kt` - JNI 接口

**职责**:
- 加载原生库
- 管理 sysroot
- 提供编译接口
- 错误处理

### 6. JNI Layer (JNI 层)

**技术栈**: C++ + Clang/LLVM

**主要组件**:
- `native_compiler.cpp` - 编译器实现

**职责**:
- 实现编译逻辑
- 调用 Clang/LLVM
- 执行构建工具
- 返回编译结果

## 🔄 编译流程

### 单文件编译流程

```
用户点击编译
    ↓
CompileProjectUseCase.execute()
    ↓
检测项目类型 (单文件)
    ↓
SingleFileCompiler.compile()
    ↓
NativeCompiler.emitObj()
    ↓
native_compiler::emitObj() (JNI)
    ↓
Clang Frontend 解析
    ↓
LLVM Backend 生成代码
    ↓
返回 .o 文件
```

## 📦 资源管理

### Sysroot 结构

```
sysroot/
├── usr/
│   ├── bin/                    # 工具二进制
│   ├── include/                # 系统头文件
│   │   ├── c++/v1/            # C++ 标准库头文件
│   │   ├── aarch64-linux-android/
│   │   └── *.h
│   ├── lib/                    # 系统库
│   │   └── <triple>/
│   │       ├── <api>/         # API 级别特定库
│   │       │   ├── libc.so
│   │       │   ├── libm.so
│   │       │   └── ...
│   │       └── runtime/       # 运行时库
│   │           ├── libLLVM-17.so
│   │           ├── libclang-cpp.so
│   │           └── libc++_shared.so
│   └── share/                  # 共享数据
└── lib/
    └── clang/17/include/      # Clang 资源头文件
```


## 🎯 设计原则

### SOLID 原则应用

1. **Single Responsibility (单一职责)**
   - 每个类只负责一件事
   - 例如: `NativeLoader` 只负责加载库

2. **Open/Closed (开闭原则)**
   - 对扩展开放，对修改关闭
   - 通过插件系统扩展功能

3. **Liskov Substitution (里氏替换)**
   - 子类可以替换父类
   - 例如: 不同的编译器实现相同接口

4. **Interface Segregation (接口隔离)**
   - 接口应该小而专注
   - 避免臃肿的接口

5. **Dependency Inversion (依赖倒置)**
   - 依赖抽象而非具体实现
   - 使用接口和抽象类

### 其他原则

- **KISS**: 保持简单
- **YAGNI**: 只实现需要的功能
- **DRY**: 避免重复代码

## 🔐 安全考虑

### SELinux 限制

Android 10+ 的 SELinux 策略限制应用沙箱内执行可执行文件。

**解决方案**:
- 将所有工具编译为共享库 (.so)
- 通过 JNI 调用而非 exec()
- 详见 [Android SELinux 解决方案](Android-SELinux-解决方案.md)

### 权限管理

- 存储权限: 读写项目文件
- 网络权限: 下载依赖包（未来）
