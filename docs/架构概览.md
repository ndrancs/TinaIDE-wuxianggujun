## 核心模块

### 1. UI Layer (界面层)

**技术栈**: Android Fragment + Material Design 3

**主要组件**:
- `MainActivity.kt` - 主活动
- `ui/fragment/` - 各个界面 Fragment
- `ui/dialog/` - 对话框组件
- `ui/adapter/` - 列表适配器

### 2. Editor Layer (编辑器层)

**技术栈**: Sora Editor + Tree-sitter

**主要组件**:
- `EditorFragment.kt` - 编辑器容器
- `editor/language/` - 语言支持（C/C++/CMake）
- `editor/TinaDarkColorScheme.kt` - 主题配色

### 3. LSP Service (语言服务)

**技术栈**: clangd + SimpleLspClient

**主要组件**:
- `lsp/LspService.kt` - LSP 服务入口
- `lsp/LspRequestDispatcher.kt` - 请求调度
- `lsp/LspResultCache.kt` - 结果缓存
- `lsp/simple/simple_lsp_client.cpp` - C++ 客户端

### 4. Compiler (编译器)

**技术栈**: Clang/LLVM + LLD

**主要组件**:
- `core/compile/CompileProjectUseCase.kt` - 编译用例
- `core/nativebridge/NativeCompiler.kt` - JNI 接口
- `cpp/compiler/clang_compiler.cpp` - Clang 编译器
- `cpp/linker/lld_linker.cpp` - LLD 链接器

### 5. Native Bridge (原生桥接)

**主要组件**:
- `NativeLoader.kt` - 库加载器
- `SysrootInstaller.kt` - Sysroot 安装器
- `AbiResolver.kt` - ABI 解析

## 编译流程

```
用户代码 (.cpp)
    │
    ▼
CompileProjectUseCase
    │
    ▼
NativeCompiler.emitObj()
    │
    ▼
clang_compiler (C++)
    │
    ├─► 词法/语法分析
    ├─► 语义分析
    └─► 生成目标文件 (.o)
        │
        ▼
NativeCompiler.linkSo()
    │
    ▼
lld_linker (C++)
    │
    └─► 生成共享库 (.so)
        │
        ▼
NativeCompiler.runSharedIsolated()
    │
    └─► fork + dlopen 执行
```

## LSP 流程

```
用户编辑代码
    │
    ▼
LspEditorBinding.didChange()
    │
    ▼
LspService.requestCompletion()
    │
    ▼
SimpleLspClient (JNI)
    │
    ▼
pipe → clangd
    │
    ▼
JSON 响应
    │
    ▼
补全列表显示
```

## 资源管理

### Sysroot 结构

```
sysroot/
├── usr/
│   ├── include/                # 系统头文件
│   │   ├── c++/v1/            # C++ 标准库
│   │   └── *.h
│   └── lib/
│       └── <triple>/
│           ├── <api>/         # API 级别特定库
│           └── runtime/       # 运行时库
│               ├── libLLVM-17.so
│               ├── libclang-cpp.so
│               ├── libclangd.so
│               └── libc++_shared.so
└── lib/
    └── clang/17/include/      # Clang 资源头文件
```

## 设计原则

- **KISS**: 保持简单
- **YAGNI**: 只实现需要的功能
- **DRY**: 避免重复代码
- **SOLID**: 面向对象设计原则
